# GitHub outreach issue template

Use this only for repositories where a lightweight correction/feedback issue is appropriate. If a project asks not to receive this type of issue, stop and use email or no contact.

## Title

```text
Request for correction/feedback: OpenSciFlow related-project entry and workflow metadata
```

## Body

```text
Hello [Project] maintainers,

I am Kaining Liu, one of the initiators of OpenSciFlow, an early open initiative for modular, reproducible, local-first AI for Science workflows.

We listed [Project] as a related project in our early AI for Science workflow landscape:

[link to entry or repo]

We are not claiming any partnership. The goal is to map existing tools accurately and learn what metadata a workflow system should record before running or citing scientific tools.

Could you help us check two small points?

1. Is our description of [Project] accurate?
2. For a workflow system that may call or cite [Project], what metadata should be required? For example: inputs/outputs, environment, container support, HPC/Slurm notes, validation tests, license, citation, known limitations, or safety notes.

Relevant OpenSciFlow draft links:

- Landscape list: https://github.com/OpenSciFlow/awesome-ai4s-workflows
- Plugin manifest draft: https://github.com/OpenSciFlow/plugin-manifest
- Workflow templates draft: https://github.com/OpenSciFlow/workflow-templates

If you prefer that we revise or remove the entry, please say so and we will update it.

Thank you.
```

## Rules

- Do not open duplicate issues.
- Do not use words like partner, collaboration, official integration, endorsed, supported, or certified.
- Do not ask maintainers to install or adopt OpenSciFlow.
- Keep the request correction-focused and low burden.
